# Timelapse — Flutter App — Development Guide

## Project overview
Long-delay photo timelapse creator for Android (API 29+). Users create projects, get interval-based reminders, align shots with a ghost overlay, and export animations. Freemium model. Phase 1 is fully on-device; cloud rendering is Phase 2.

## Data models

### Project (Drift table `projects`)
| Column | Type | Notes |
|---|---|---|
| id | int PK | auto-increment |
| name | String | 1–100 chars |
| createdAt | DateTime | default now |
| intervalSeconds | int | default 86400 (24h) |
| gpsEnabled | bool | default false |
| referencePhotoId | int? | no FK (avoids circular ref) |
| folderPath | String | `<appDocs>/projects/<id>/photos/` |
| isActive | bool | default true |
| notificationsEnabled | bool | default true |
| lastPhotoAt | DateTime? | updated after each capture |

### Photo (Drift table `photos`)
| Column | Type | Notes |
|---|---|---|
| id | int PK | auto-increment |
| projectId | int FK | → projects.id |
| filePath | String | absolute path to JPEG |
| takenAt | DateTime | default now |
| latitude | double? | nullable |
| longitude | double? | nullable |

### File storage
Photos stored at: `<appDocumentsDirectory>/projects/<projectId>/photos/<timestamp>.jpg`
Exports stored at: `<appDocumentsDirectory>/exports/`
**Never DCIM** — uses app-scoped storage, no storage permission needed on Android 10+.

## Coding conventions

### Flutter / Dart
- Use `withValues(alpha: x)` — **not** `.withOpacity(x)` (deprecated in Flutter 3.41)
- Use `CardThemeData(...)` — **not** `CardTheme(...)` (renamed in Flutter 3.41)
- Use `DropdownButtonFormField(initialValue: ...)` — **not** `value:` (deprecated after v3.33)
- Wrap non-required Drift companion fields in `Value<T>(...)` e.g. `Value(intervalSeconds)`
- State management: Riverpod providers. No `setState` in business logic — only in local UI state.
- All screens are `ConsumerWidget` or `ConsumerStatefulWidget`.
- No `BuildContext` use after `await` without `mounted` guard.

### File naming
- `snake_case` for all files
- One screen per file, one widget class per file (unless small private helper)

### Providers pattern
```dart
// Repository provider
final projectRepositoryProvider = Provider<ProjectRepository>((ref) { ... });

// Stream provider (auto-updates UI)
final projectsStreamProvider = StreamProvider<List<Project>>((ref) { ... });

// Family provider (per-ID)
final photosStreamProvider = StreamProvider.family<List<Photo>, int>((ref, id) { ... });
```

## Freemium gating

| Feature | Free | Premium |
|---|---|---|
| Projects | ≤ 3 | Unlimited |
| Draft export (MP4/GIF/WebP) | ✅ | ✅ |
| 4K export | ❌ | ✅ |
| Color grading | ❌ | ✅ |
| Cloud HQ render | ❌ | ✅ (Phase 2) |

- `kFreeProjectLimit = 3` in `project_repository.dart`
- `canCreateProject()` always checks count < 3 (premium always false in Phase 1)
- 4th project attempt → redirects to `/premium`
- RevenueCat (`purchases_flutter`) is **not** in pubspec for Phase 1 — add it for Phase 2

## Notification logic

```
User sets interval = 4h
→ lastPhotoAt stored in DB after each capture
→ WorkManager fires every 15 min
→ Calculates (lastPhotoAt + interval) - now = remaining
→ Updates persistent notification: "Next photo in: 3h 42m"
→ When remaining ≤ 0: "Time to take your photo! 📷"
→ Tap notification → deep-link → CameraScreen
```

- One notification per project (uses `projectId` as notification ID)
- Channel: `timelapse_reminder` — silent, no sound/vibration, `ongoing: true`
- Call `NotificationService.showCountdownNotification(...)` after every capture
- Call `SchedulerService.rescheduleForProject(project)` after saving settings

## Export (FFmpeg)

`ExportService.export(ExportJob, onProgress)` builds FFmpeg commands:

- **MP4**: `libx264`, scale 1280px wide, `yuv420p`, faststart
- **GIF**: palette-based, scale 640px wide
- **WebP**: `libwebp_anim`, scale 1280px wide, quality 80

Photos are copied to a temp dir as `0001.jpg`, `0002.jpg`, ... before FFmpeg runs.
Output saved to `<appDocs>/exports/`. Share via `share_plus`.

`ffmpeg_kit_flutter_min` is discontinued upstream but functional for Phase 1. For Phase 2 consider switching to `ffmpeg_kit_flutter` full or a server-side approach.

1. **RevenueCat** — add `purchases_flutter` to pubspec, activate in `PremiumScreen`, wire `canCreateProject()` to real entitlement check
2. **Firebase Auth + Firestore** — user accounts, project sync

6. **AlarmManager platform channel** — 1-minute precision countdown (vs WorkManager's 15-min minimum)

## Known limitations (Phase 1)

- WorkManager minimum interval is 15 minutes — notification countdown updates at most every 15 min when app is backgrounded (1-min precision requires AlarmManager via platform channel)
- `ffmpeg_kit_flutter_min` package is discontinued — monitor for breakage on future Android versions
- No crash reporting / analytics wired up yet
- No widget tests beyond placeholder — write integration tests before Phase 2 release
